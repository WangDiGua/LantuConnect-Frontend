# LantuConnect 统一智能接入平台

## 平台定位

LantuConnect（兰智通）是面向高校的**注册网关型**统一智能能力接入平台。
核心职责：注册、发现、路由、鉴权、健康监控、质量度量。Agent/Skill 本身在外部系统中运行。

## 三类能力提供方

- **internal（公司产品）**：低代码平台、工作流生产工具、代码 Agent、页面型智能应用
- **partner（第三方校园服务商）**：给学校提供服务的合作公司
- **cloud（云智能能力）**：阿里云百炼、火山引擎等在线 AI 服务商

## 整体架构

```
能力消费方                  统一智能接入平台                    能力提供方
─────────              ─────────────────────              ─────────
校园门户/APP  ──┐        ┌──────────────────┐         ┌── 低代码平台
业务系统    ──┤  ──→   │  接入网关          │   ──→   ├── 工作流引擎
第三方调用  ──┘        │  注册中心          │         ├── 代码Agent
                      │  权限引擎          │         ├── 页面型应用
                      │  健康检查+熔断降级  │         ├── 校园服务商
                      │  质量度量          │         ├── 阿里云百炼
                      │  智能路由          │         └── 火山引擎
                      └──────────────────┘
```

## 两种调度模式

1. **API 直调**：业务系统通过 `agent_code` 精准调用指定能力，适合后端集成
2. **LLM 智能路由**：主 Agent 根据用户意图自动选择 Tool，适合对话场景

## 接入协议

REST / MCP / OpenAPI / Webhook / SSE / 容器化

后端通过协议适配器统一转换为内部协议：
```
ProtocolAdapter → InternalRequest/Response → RuntimeRole (TOOL / SUB_AGENT / PAGE)
```

## 关键设计决策

1. **不单独管理 Skill 表**：Skill/Tool 统一在 `genie_external_agent` 中管理，`mode=TOOL` 即为技能
2. **MCP Server 层级管理**：通过 `parent_id` 字段实现 MCP Server → Tool 父子关系
3. **智能应用 = PAGE_APP 类型 Agent**：独立前端应用注册后通过 iframe/微前端嵌入

## source_type 枚举

| 值 | 含义 |
|---|---|
| `platform_builtin` | 平台内置能力（知识库搜索、任务调度） |
| `lowcode` | 低代码平台产出的 Agent |
| `workflow` | 工作流引擎编排的 Agent |
| `code_agent` | 公司自研代码 Agent |
| `page_app` | 页面型智能应用 |
| `mcp_cloud` | 云 MCP 服务（阿里云、火山引擎） |
| `http_partner` | 第三方校园服务商 REST API |
