# 后端架构

## 技术栈

Java 17 + Spring Boot 3.2.x + Spring Cloud Gateway + Spring WebFlux + Spring Security + MyBatis Plus + Resilience4j

## 服务拆分

| 服务 | 职责 |
|---|---|
| gateway-service | 协议适配、路由转发、流量治理 |
| agent-service | Agent 注册、版本管理、生命周期 |
| security-service | 认证授权、AI 防火墙、数据脱敏 |
| billing-service | Token 计量、成本计算、账单生成 |

## 协议适配层

所有外部协议通过 `ProtocolAdapter` 接口统一转换为 `InternalRequest/Response`：

```java
public interface ProtocolAdapter {
    String getProtocolType();    // mcp / rest / openapi / webhook / sse
    Mono<InternalRequest> adaptRequest(ServerWebExchange exchange);
    Mono<Void> adaptResponse(ServerWebExchange exchange, InternalResponse response);
}
```

## 运行时角色映射

由 `ExternalAgentRuntimeResolver` 根据 agent_type + mode 决定：

| agent_type | 默认角色 | mode=TOOL 时 | mode=SUBAGENT 时 | mode=PAGE_APP 时 |
|---|---|---|---|---|
| mcp | SUBAGENT | TOOL | SUBAGENT | - |
| http_api | TOOL | TOOL | SUBAGENT | PAGE |
| builtin | TOOL | TOOL | SUBAGENT | - |

## 外部能力发现

- `CloudMcpDiscoveryService`：从阿里云百炼、火山引擎发现 MCP Server
- `ExternalAgentService`：加载、排序、注册、同步外部 Agent
- `TaskDispatchTool`：子 Agent 调度入口，根据 RuntimeRole 分发

## 请求处理链

```
请求 → 协议适配 → JWT认证 → 租户隔离 → AI防火墙 → 限流 → 熔断判断
     → 语义缓存查询 → (命中则返回) → 路由到目标Agent → 数据脱敏 → 响应
```

## 流量治理

- **限流**：Redis 令牌桶，支持 APP/USER/IP/AGENT/DEPARTMENT 五个维度
- **熔断**：Resilience4j，failure_threshold + 滑动窗口 → open → half_open → closed
- **降级**：fallback_agent_id 指定备用 Agent，或返回 fallback_message
- **语义缓存**：Milvus 向量相似度检索，阈值 0.95，命中直接返回

## API 规范

- 基础路径：`/api/v1/`
- 统一响应：`{ code: int, message: string, data: T }`
- 分页：`{ records: T[], total, pageNum, pageSize }`
- 认证：`Authorization: Bearer {token}`
- 链路追踪：每个请求携带 `X-Trace-Id`
