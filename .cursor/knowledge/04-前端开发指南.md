## 路由约定

控制台路径格式：`/c/{admin|user}/{sidebar}/{sub}`

- Admin 视角：系统概览、系统配置、用户与权限、模型与算力、工具与生态、运营与安全、集成与中台、观测与告警、数据与存储、审计与日志
- User 视角：工作台、AI 助手、Agent 与编排、工作流、资源与知识、模型与推理、工具与插件、发布与连接、数据与评测、用量与账单、账户与偏好、文档与教程

路由辅助函数在 `constants/consoleRoutes.ts`：`buildConsolePath()` / `parseConsoleRole()` / `isValidConsolePath()`

中英文映射在 `constants/routeMapping.ts`：`toEnglishRoute()` / `toChineseLabel()`

## 状态管理

- **Zustand**（`stores/`）：auth 状态、全局 app 状态
- **React Context**（`context/`）：`UserRoleContext`（admin/user 切换）、`LayoutChromeContext`
- **React Query**（`hooks/`）：服务端数据的获取、缓存、同步

## 核心 DTO 类型

定义在 `types/dto/` 下：

| 文件 | 核心类型 |
|---|---|
| agent.ts | Agent, AgentCreatePayload, AgentVersion, AgentMarketItem |
| tool.ts | ToolItem, McpServer, McpTool, ToolReview |
| knowledge.ts | KnowledgeBase, KBDocument |
| workflow.ts | Workflow, WorkflowNode, WorkflowEdge, WorkflowRun |
| asset.ts | PromptTemplate, TermEntry, SecretEntry |

## API 服务层约定

在 `api/services/` 下按领域拆分，使用 `lib/http`（Axios）发请求：

- `agent.service.ts`：list, getById, create, update, delete, sendDebugMessage
- `tool.service.ts`：discover, listMine, createMcpServer
- `knowledge.service.ts`：list, getById, create, uploadDocument, hitTest
- `workflow.service.ts`：list, create, execute, listRuns
- `monitoring.service.ts`：KPIs, callLogs, alerts, traces
- `admin.service.ts`：resources, stats, health, gatewayRoutes
