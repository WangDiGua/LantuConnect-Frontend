# 安全、监控与部署

## 安全架构

### 认证授权

- JWT + API Key 双模认证
- RBAC 权限模型：角色 → 权限 → 资源+操作
- 租户隔离：corp_id + client_code 维度

### 请求安全链

```
Token解析 → 签名验证 → 权限校验 → 租户隔离 → AI防火墙 → 注入检测 → PII脱敏 → 业务处理 → 脱敏还原 → 响应
```

### AI 防火墙

- Prompt 注入防护（正则 + 语义检测）
- 越狱攻击检测
- PII 信息泄露防护
- 敏感词过滤

### 数据脱敏

| 类型 | 策略 |
|---|---|
| 手机号 | 保留前3后4：`138****5678` |
| 邮箱 | 保留前2+域名：`zh****@example.com` |
| 身份证 | 保留前6后4：`110108****1234` |
| 银行卡 | 保留前4后4 |

## 监控体系

### 指标采集

- Prometheus 采集 JVM、QPS、延迟、错误率
- 自定义指标：Token 消耗、缓存命中率、Agent 健康状态
- Grafana 可视化

### 链路追踪

- SkyWalking Agent 自动注入
- 每个请求携带 X-Trace-Id
- 全链路：网关 → 认证 → 限流 → 缓存 → Agent → 响应

### 告警规则

| 指标 | 阈值 | 级别 |
|---|---|---|
| 错误率 > 5% | 持续 5 分钟 | ERROR |
| P99 延迟 > 2s | 持续 5 分钟 | WARNING |
| Agent 健康异常 | 立即 | CRITICAL |
| Token 消耗异常 | 速率 > 10万/h | WARNING |

### 日志

- ELK Stack 收集
- 调用日志保留 30 天，归档 90 天
- 审计日志保留 180 天，归档 3 年
- Token 使用记录按月分区

## 高可用

- 目标：99.9% SLA
- 限流 + 熔断 + 降级三级防护
- Redis 缓存多级：本地 → Redis → Milvus
- 数据库主备 + 自动备份

## 部署

### 环境

| 环境 | 配置 |
|---|---|
| 开发 | 4核8G × 2 |
| 测试 | 8核16G × 3 |
| 生产 | 16核32G × 3+ |

### 容器化

- Kubernetes 集群部署
- Gateway 服务 3 副本起步，HPA 自动扩缩（CPU 70%）
- Systemd 管理服务进程
- Nginx 反向代理 + 负载均衡

### 前端部署

- Vite 构建输出到 `dist/`
- Nginx 托管静态资源 + API 反向代理
